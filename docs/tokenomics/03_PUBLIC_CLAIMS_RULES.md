# Public Claims Rules

Status: Accepted public-communication policy

## Allowed Status Labels

- Draft
- Planned
- Research
- Private Devnet
- Public Testnet
- Mainnet Candidate
- Coming Soon
- Prototype
- Experimental

## Forbidden Claims Without Verified Implementation

- Mainnet live
- Wallet live
- Explorer live
- Mining pool live
- DAO live
- Staking rewards live
- Marketplace live
- Guaranteed yield
- Exchange ready
- Investment opportunity
- Guaranteed returns

## Tokenomics Messaging Rules

- Do not present VIRE as guaranteed investment.
- Do not promise exchange listings.
- Do not promise staking or treasury behavior before models exist.
- Describe the base-fee burn plus miner-tip transfer model as implemented
  candidate behavior, not as final public-Mainnet economics.
- Keep contract gas and allocation/treasury behavior explicitly unresolved.

## Cross-System Impact

- Website/Admin: public copy must follow these rules directly.
- Docs: whitepaper, roadmap and tokenomics docs must stay aligned.
- Wallet, Explorer, RPC and Indexer: UI labels must reflect the actual verified environment only.
